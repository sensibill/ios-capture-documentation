Feature Flags

Property Description Default Value
enableBlurDetection Enables a blur warning on the preview screen after capturing (Note: this feature is only available on the iPhone 6s and newer) true
enableFeedbackToast Enables feedback toast on the capture screen true
enableAutoCapture Enables the auto-capture feature true
enableImageGallery Enables the user to select an image from their phone’s image gallery true
enableAddPage Enables the user to create multi-page receipts true
enableLongCapture Enables the long receipt capture feature - capturing a long receipt with a top and bottom image true
enableCrop Enables the ability to crop captured images true
enableFlash Enables the ability to toggle the phone’s flash on/off to help capture in low-light situations true
defaultToAutoCapture Starts the capture screen with auto-capture mode enabled by default true
attachLocationData Enables the ability to attach the device’s current location to the image being captured. Requires location permission to have been granted by the user before launching capture. Location permission requires the key NSLocationWhenInUseUsageDescription to be include in your application’s Info.plist false

Capture Colors

Colors and configs contained in the CaptureTheme class are ones that apply to specific UI elements in a specific section of the module. For example, the done button on the crop screen and capture preview screen are both Primary Buttons, so to make one differ visually from the other you would have change one of their corresponding colors/configs in the CaptureTheme module, as changes to primary buttons in the DefaultTheme will apply to them both. A CaptureTheme can be initialized with a DefaultTheme to provide default color mappings different from the set defaults.

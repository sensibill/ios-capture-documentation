Installation

Overview

The Sensibill Capture framework is available in a private repository on bitbucket. Please contact Sensibill to obtain access to the repository.

Once access is obtained, follow the instructions below to use the static or dynamic framework.

Via CocoaPods

CocoaPods provides a quick way to download external dependencies for an iOS project. The Sensibill Capture Module can be downloaded and accessed via CocoaPods:

  1. If your project isn’t using CocoaPods already, follow the set up guide from the official CocoaPods website here

  2. Once your project is set up with CocoaPods, add the Sensibill dependency to your project’s Podfile in the following format:

  • To use the dynamic framework:

pod 'Sensibill/Capture', :git => 'git@bitbucket.org:sensibill/sensibill-ios-capture-release.git', :tag => 'vX.Y.Z', where vX.Y.Z represents to version of the module you want to use.

  • To use the static framework:

pod 'Sensibill/CaptureStatic', :git => 'ssh://git@bitbucket.org/sensibill/sensibill-ios-capture-release.git', :tag => 'vX.Y.Z', where vX.Y.Z represents to version of the module you want to use.

  1. In your terminal, navigate to the root directory of your application’s project folder.

  2. Run pod install.

Direct Download

  1. Download a zip archive named vX.Y.Z.zip from the downloads section of Bitbucket (where X.Y.Z is the version of the Capture, for example, 1.26.3).

  2. Unarchive the zip file. Drag and drop SensibillCapture.framework file into your project, and ensure that it is included in Embedded Binaries and Linked Frameworks and Libraries in your project settings.

  3. Create a new Run Script Phase in your target’s Build Phases underneath the Embed Frameworks phase. Input the following command in the text field of the new phase:

bash "$BUILT_PRODUCTS_DIR/$FRAMEWORKS_FOLDER_PATH/SensibillCapture.framework/strip-unused-architectures.sh"
  1. If using Xcode 12.3 or above, update the Validate Workspace setting to Yes. This is to avoid one of the following compilation errors:

Building for the iOS Simulator, but the linked and embedded framework ‘SensibillCapture.framework’ was built for iOS + iOS Simulator.

or

Building for iOS, but the linked and embedded framework ‘SensibillCapture.framework’ was built for iOS + iOS Simulator.

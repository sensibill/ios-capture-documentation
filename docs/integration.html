Sensibill Capture - iOS

Sensibill Capture is a standalone framework or static library that allows for the scanning and cropping of receipts and documents. The module can scan and return up to three documents simultaneously to the integrator’s app.

Requirements

  • iOS 11.0 +
  • Xcode 11.0 +
  • Cocoapods 1.9.3

Installation

You can use CocoaPods to install SensibillCapture by adding it to your Podfile:

# To use as a dynamic framework (Recommended)
pod 'SensibillCapture', :git => 'git@github.com:sensibill/sdk-ios-capture.git'

# To use as a static library
pod 'SensibillCaptureStatic', :git => 'git@github.com:sensibill/sdk-ios-capture.git'

Usage

The module is presented modally as the first view is a camera preview. To initialize and present the default module use the following:

import SensibillCapture
// or SensibillCaptureStatic if using the static library option

...

class YourClass {

...

    let captureNavigationController = CaptureNavigationController()
    captureNavigationController.modalPresentationStyle = .fullScreen
    captureNavigationController.captureDelegate = self
    present(captureNavigationController, animated: true, completion: nil)

...

}

// Implement the protocol method
extension YourClass: CaptureNavigationControllerDelegate {

    // This will be called when the module is complete
     func captureNavigationController(_ controller: CaptureNavigationController, didCapture images: [UIImage]) {
        // One to Three photos will be returned
        print("Received \(images.count) photos")
    }

}

Configuration

Overriding Images, Colors, Strings, & Configurations

  1. Add your images and colors to an assets folder called SensibillAssets.xcassets in your project or in a custom bundle. The color or image should have the same name as the one you’re overriding.
  2. Add your configurations to a plist named SensibillConfiguration.plist in your project or in a custom bundle. The key name should be the same as the one you’re overriding.
  3. Specify keys for strings you’d like to override in a Localizable.strings file.
  4. Initialize the module with the bundle that contains your SensibillAssets.xcassets, SensibillConfiguration.plist, and/or Localizable.strings files. This is commonly your main application bundle but you may choose to create a new resource bundle.

Example

let bundle = Bundle.main
// or if creating a new resource bundle in your project:
let bundle = Bundle.main.path(forResource: "MyBundleName", ofType: "bundle")

let captureConfig = CaptureConfiguration(bundle: bundle)
let captureNavigationController = CaptureNavigationController(config: captureConfig)

Note: If you are not using your main application bundle and you are including custom colors in the bundle, you will need to compile your assets using the following run script:

PATH_TO_BUNDLE=${PROJECT_DIR}/Custom.bundle/
PATH_TO_ASSETS=${PROJECT_DIR}/Custom.bundle/SensibillAssets.xcassets
xcrun actool ${PATH_TO_ASSETS} --compile ${PATH_TO_BUNDLE} --platform iphoneos --minimum-deployment-target ${IPHONEOS_DEPLOYMENT_TARGET}

Overriding Colors Programatically

Swift

let bundle = Bundle.main
// or if creating a new resource bundle in your project:
let bundle = Bundle(url: Bundle.main.path(forResource: "MyBundleName", ofType: "bundle")!)

// Create Defaults
// Colors defined in the default theme will have module wide effects
let defaultTheme = DefaultTheme(overrideBundle: customBundle)
defaultTheme.uiSecondaryBackground = .purple

// Create Capture theme with defaults
// Colors defined in the capture theme will customize individual components, overriding colors provided by the default theme
let captureTheme = CaptureTheme(defaults: defaultTheme)
captureTheme.cropDoneButtonBackgroundColor = .green

// Create and pass configs to the capture module
let captureConfig = CaptureConfiguration(theme: captureTheme)
let capture = CaptureNavigationController(config: captureConfig)

// Present it
capture.modalPresentationStyle = .fullScreen
capture.captureDelegate = self
present(capture, animated: true, completion: nil)

Objective-C

// Get configuration bundle (optional)
NSBundle *customBundle = [NSBundle mainBundle];
// or if creating a new resource bundle in your project:
NSBundle *customBundle = [NSBundle bundleWithPath: [[NSBundle mainBundle] pathForResource:@"CustomBundle" ofType:@"bundle"]];

// Create Defaults
SBLDefaultTheme *defaults = [[SBLDefaultTheme alloc] initWithOverrideBundle:customBundle];
defaults.uiSecondaryBackground = [UIColor purpleColor];

// Colors defined in the default theme will have module wide effects

// Create Capture theme with defaults
SBLCaptureTheme *captureTheme = [[SBLCaptureTheme alloc] initWithDefaults:defaults];
captureTheme.capturePreviewDoneButtonBackgroundColor = [UIColor greenColor];

// Colors defined in the capture theme will customize individual components, overriding colors provided by the default theme

// Create Config with capture theme
SBLCaptureConfiguration *config = [[SBLCaptureConfiguration alloc] initWithTheme:captureTheme features:nil];

// Create and pass configs to the capture module
SBLCaptureNavigationController *capture = [[SBLCaptureNavigationController alloc] initWithConfig:config];

// Present it
capture.modalPresentationStyle = UIModalPresentationFullScreen;
[self presentViewController:capture animated:YES completion:nil];

Configuring Fonts Programmatically

The DefaultTheme object has a property called fonts that allows the integrator to override the three default font styles (Regular, SemiBold, and Bold) via the defaults property by passing a FontSet. The integrator also has more fine-grained control and can override the ten font styles used throughout the module:

  • body
  • subhead
  • headline
  • largeTitle
  • title
  • titleTwo
  • titleThree
  • footnote
  • callout
  • captionOne
  • captionTwo

Example

let myRegularFont = UIFont(name: "Zapfino", size: 12)
let mySemiboldFont = UIFont(name: "Georgia-Regular", size: 12) // optional, will be mapped to bold if not provided
let myBoldFont = UIFont(name: "Georgia-Bold", size: 12)

let myFontSet = FontSet(regular: myRegularFont, semibold: mySemiboldFont, bold: myBoldFont)

let defaultTheme = DefaultTheme(overrideBundle: customBundle)
// Override the default fonts that are mapped to the 10 font styles
defaultTheme.fonts.defaults = myFontSet
// Override a specific font style and size
defaultTheme.fonts.titleThree = UIFont.systemFont(ofSize: 50)

Configuring Feature Flags

The capture module provides the object CaptureFeatures, which holds all of its feature flags. Create an instance of this object and then access and set the flags as properties of the object. Finally, pass the object to your CaptureConfiguration object that you’re using to initialize the module.

Feature Flags

Property Description Default Value
enableBlurDetection Enables a blur warning on the preview screen after capturing (Note: this feature is only available on the iPhone 6s and newer) true
enableFeedbackToast Enables feedback toast on the capture screen true
enableAutoCapture Enables the auto-capture feature true
enableImageGallery Enables the user to select an image from their phone’s image gallery true
enableAddPage Enables the user to create multi-page receipts true
enableLongCapture Enables the long receipt capture feature - capturing a long receipt with a top and bottom image true
enableCrop Enables the ability to crop captured images true
enableFlash Enables the ability to toggle the phone’s flash on/off to help capture in low-light situations true
defaultToAutoCapture Starts the capture screen with auto-capture mode enabled by default true

Example

let features = CaptureFeatures()
features.enableFlash = false
features.enableImageGallery = false
features.defaultToAutoCapture = false
features.enableCrop = false
features.enableAddPage = false

let captureConfig = CaptureConfiguration(bundle: customBundle, features: features)

// Then initialize and present the module
